## Component Inventory

Components to be built in this phase:

### `<CardGrid />`
- CSS grid: `grid-cols-2` on mobile (375px+), `grid-cols-4` on desktop (1280px+)
- Grid gap: `md` (16px) on mobile, `lg` (24px) on desktop
- AnimatePresence wraps the grid so dismissed cards animate out and remaining cards reflow with Framer Motion layout animation
- Background: `cyber-black`

### `<CelebrityCard />`
- Dimensions: fixed aspect ratio — 3:4 portrait on mobile, 2:3 on desktop
- Background: `cyber-panel` (`#1a1a2e`)
- Border: 1px solid `neon-cyan` at 40% opacity at rest; full `neon-cyan` opacity on hover/drag
- Box shadow at rest: `0 0 8px rgba(0,245,255,0.15)`
- Box shadow on hover: `0 0 16px rgba(0,245,255,0.35)`
- Border radius: 8px
- Framer Motion `drag="x"` enabled; `dragElastic: 0.15`; `dragConstraints: { left: 0, right: 0 }` — card snaps back unless threshold met
- Rotate during drag: tilt up to ±15deg proportional to drag distance (Framer Motion `useMotionValue` + `useTransform`)
- Dismiss threshold: 120px horizontal drag OR velocity > 500px/s

Layout inside card (top to bottom):
1. Profile photo — full-width, top-aligned, 60% of card height, `object-cover`
2. Card body — remaining 40% of card height, `padding: md` (16px)
   - Display name — 16px, semibold, `cyber-text`
   - @handle — 12px, semibold, `neon-cyan`
   - Bio snippet — 14px, regular, `cyber-text`, max 2 lines, `line-clamp-2`
   - Follower count — 12px, regular, `cyber-muted`

### `<DismissButton />`
- Position: absolute top-right corner of card — `top: 8px; right: 8px`
- Size: 44px × 44px touch target (visually ~28px circle with padding)
- Background: `cyber-black` at 80% opacity
- Icon: × (neon-pink, 18px)
- Hover: border `neon-pink` 1px, box-shadow `0 0 8px rgba(255,0,110,0.5)`
- Focus ring: 2px `neon-cyan` offset outline
- Not rendered on @chadcluff card (`isChad: true`)

### `<StickyHeader />`
- `position: sticky; top: 0; z-index: 50`
- Background: `cyber-black` with `backdrop-blur` (so cards visible scrolling behind)
- Height: 48px
- Content: "Twitter Celebrity" — 28px, bold, `neon-cyan`, centered
- Bottom border: 1px solid `neon-cyan` at 20% opacity

### `<InstructionHint />`
- Positioned: centered below header, above grid
- Text: "Tap × to eliminate" — 12px, regular, `cyber-muted`
- Animation: fade in on mount, fade out after first card dismiss or after 4 seconds, whichever comes first
- Does not re-appear after fade

## Interaction Contract

### Staggered Entry Animation
- Cards appear in sequence, not simultaneously
- Delay per card: `index * 60ms`
- Each card animates: `opacity: 0 → 1`, `y: 20px → 0`
- Duration: 300ms per card, `easeOut` easing
- Total entry for 15 cards: ~1200ms

### Card Drag / Swipe Dismiss
- Drag starts immediately on `pointerdown` (no delay)
- During drag: card tilts ±15deg, edge facing dismiss direction gets `neon-pink` tint overlay at 20% opacity
- At threshold (120px or velocity > 500px/s): card exits with `x: ±600px, opacity: 0, rotate: ±30deg` — duration 250ms
- Below threshold: card snaps back with spring physics (`stiffness: 400, damping: 40`)
- `touch-action: none` on card element to prevent scroll conflict during drag (see PITFALLS.md)

### X Button Tap Dismiss
- On click/tap: same exit animation as swipe dismiss — card exits toward right (`x: 600px`) by default
- Framer Motion `exit` variant fires; AnimatePresence removes card from DOM after animation

### Grid Reflow
- Remaining cards animate to new positions with Framer Motion `layout` prop
- Reflow duration: 300ms, `spring` physics (`stiffness: 300, damping: 35`)
- No jarring snaps — smooth slide into vacated gaps

### @chadcluff Card
- No `<DismissButton />` rendered
- Drag is disabled (`drag={false}`)
- Border uses `neon-yellow` (`#f5e642`) instead of `neon-cyan` to visually distinguish
- No dismiss interaction possible — card persists

### Game State Transition
- When `dismissedCount === 14` (all non-chad cards dismissed), game state transitions to `"reveal"`
- No visual change in Phase 2 — Phase 3 owns the reveal animation
- State flag must be trackable by Phase 3 (expose via `useReducer` dispatch or context)
